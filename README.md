# Stock Tracker API

Node.js backend API for the Stock Tracker demo app. Express + TypeScript + PostgreSQL + Drizzle ORM.

## Tech Stack

- **Express** with TypeScript
- **PostgreSQL** via `pg` (node-postgres)
- **Drizzle ORM** for type-safe database queries and schema management
- **Zod** for request validation
- **JWT** for authentication
- **bcryptjs** for password hashing

## Prerequisites

- Node.js 18+
- PostgreSQL running locally

## Getting Started

1. **Install dependencies:**

   ```bash
   npm install
   ```

2. **Configure environment:**

   ```bash
   cp .env.example .env
   ```

   Edit `.env` if your PostgreSQL connection differs from the default:

   ```
   DATABASE_URL=postgresql://postgres:postgres@localhost:5432/stock_tracker
   JWT_SECRET=demo-secret-key-not-for-production
   PORT=3001
   ```

3. **Run setup (creates database + applies schema + seeds data):**

   ```bash
   npm run db:setup
   ```

   This is **idempotent** -- safe to run multiple times. It will:
   - Create the PostgreSQL database if it doesn't exist
   - Push the Drizzle schema to create tables (if they don't already exist)
   - Insert seed data only if it's missing (no duplicates)

   Seed data includes:
   - 1 demo user
   - 20 stocks with randomized prices
   - 30 days of price history per stock
   - 3 stocks (AAPL, NVDA, GOOGL) in the demo user's portfolio

4. **Start the dev server:**

   ```bash
   npm run dev
   ```

   Server runs at `http://localhost:3001`.

## API Endpoints

All endpoints except login require `Authorization: Bearer <token>` header.

| Method | Path | Description |
|--------|------|-------------|
| POST | `/api/auth/login` | Login with email/password, returns JWT |
| GET | `/api/stocks/search?q=term` | Search stocks by symbol or name |
| GET | `/api/stocks/:symbol` | Stock detail with price history and stats |
| GET | `/api/portfolio` | Get user's portfolio stocks + summary |
| POST | `/api/portfolio/stocks` | Add stock to portfolio (`{ "symbol": "TSLA" }`) |
| DELETE | `/api/portfolio/stocks/:symbol` | Remove stock from portfolio |
| GET | `/api/health` | Health check |

## Database Schema

Defined in `src/db/schema.ts` using Drizzle ORM's `pgTable` helpers:

- **users** - id, email, password_hash, name, created_at
- **stocks** - id, symbol, name, sector, current_price, daily_change, daily_change_percent, market_cap, volume, updated_at
- **portfolio_stocks** - id, user_id, stock_id, added_at (unique on user_id + stock_id)
- **stock_price_history** - id, stock_id, date, open, high, low, close, volume (unique on stock_id + date)

## Drizzle ORM

This project uses [Drizzle ORM](https://orm.drizzle.team/) for type-safe database access.

- **Schema definition:** `src/db/schema.ts` -- all tables defined with `pgTable()`
- **DB instance:** `src/db/pool.ts` -- exports both the raw `pg` Pool and the Drizzle `db` instance
- **Config:** `drizzle.config.ts` at project root
- **Migrations:** `./drizzle/` folder (generated by `drizzle-kit generate`)

Common drizzle-kit commands:

```bash
npx drizzle-kit generate   # Generate migration from schema changes
npx drizzle-kit push        # Push schema directly to database
npx drizzle-kit studio      # Open Drizzle Studio (visual DB browser)
```

## Notes

- Setup is fully automated and idempotent. Running `npm run db:setup` multiple times is safe -- it creates the database if missing, creates tables if missing, and won't duplicate data.
- All stock data is fake/randomized. No external APIs are called.
- JWT tokens expire after 24 hours.
